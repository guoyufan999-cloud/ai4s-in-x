# 编码手册（framework v2）

## 一、编码目标

本手册服务于 framework v2。当前活跃框架不再是旧的“科研工作流—合法性评价—边界协商”，而是五轴框架：

1. `discursive_context / S` 话语情境
2. `practice_position / P` 实践位置
3. `intervention_mode / I` 介入方式
4. `normative_evaluation / N` 规范评价
5. `boundary_generation / G` 边界生成

旧 A/B/C/D/E 代码仍可用于历史审计、legacy lookup 和旧 artifacts 解读，但不再作为 `iter_codebook_rows()` 的主输出。

## 二、当前活跃框架

### S. 话语情境

- `S1` 自我实践叙述
- `S2` 求助诊断
- `S3` 教程建议
- `S4` 规范争议
- `S5` 制度规则转述
- `S6` 工具服务推广
- `S7` 同行经验协商

### P. 实践位置

- `P1` 选题定位
- `P2` 文献知识组织
- `P3` 研究设计
- `P4` 数据材料
- `P5` 分析建模
- `P6` 结果核验解释
- `P7` 写作发表传播
- `P8` 组织治理评价
- `P9` 学习训练

### I. 介入方式

- `I1` 检索定位
- `I2` 摘要整理
- `I3` 草稿生成
- `I4` 润色翻译
- `I5` 分析计算
- `I6` 规划拆解
- `I7` 检查质控
- `I8` 自动化替代
- `I9` 陪练支持

### N. 规范评价

- `N0` 未评价
- `N1` 效率正当化
- `N2` 条件接受
- `N3` 可靠性担忧
- `N4` 原创/贡献担忧
- `N5` 诚信合规否定
- `N6` 责任披露要求
- `N7` 训练价值担忧
- `N8` 公平/隐私/资源风险

### G. 边界生成

- `G1` 辅助/替代分界
- `G2` 人类最终责任
- `G3` 任务风险分层
- `G4` 人工复核门槛
- `G5` 披露说明边界
- `G6` 高风险禁止
- `G7` 训练阶段保护
- `G8` 数据/知识资产边界
- `G9` 制度/期刊/学科规则边界

## 三、活跃判断规则

1. 先判断话语情境，再判断实践位置。
2. 再判断 AI 的介入方式，避免只用实践位置代替介入机制。
3. 规范评价只编码文本明确表达的接受、担忧、限制、否定或责任要求。
4. 边界生成只编码文本实际划出的规则、门槛、责任或禁止边界。
5. 允许多重编码，但不得为了填满五轴而补码。

## 四、历史字段与当前口径的关系

- 历史 A/B/C/D/E 与 legacy 字段保留为兼容层。
- `A. 科研工作流环节识别` 只作为旧框架说明保留，不是当前活跃主框架。
- 当前主输出以 [`src/ai4s_legitimacy/coding/codebook_seed.py`](../src/ai4s_legitimacy/coding/codebook_seed.py) 中的 v2 代码为准。
- `quality_v5 post-only`、正式帖子 `514`、正式评论 `0`、`comment_review_v2 deferred`、`quality_v4 historical audit` 均保持不变。

## 五、操作约束

- 任何编码都必须能回到原文证据。
- 如果文本无法支持任何五轴判断，应剔除或标为不适合当前研究问题。
- 工具推广语境可以编码为 `S6`，但只有能回到科研实践位置时才进入正式研究讨论。
- 评论层正式编码仍 deferred；不得把评论 corpus 写成当前正式评论结果。
