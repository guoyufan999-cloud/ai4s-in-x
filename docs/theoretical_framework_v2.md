# Theoretical Framework v2

## 核心转换

framework v2 将当前研究从旧的“科研工作流—合法性评价—边界协商”框架，升级为“话语情境—实践位置—介入方式—规范评价—边界生成”五轴框架。

旧框架把材料主要理解为科研环节、评价态度和边界讨论的组合。v2 更进一步追问：这些判断是在什么平台话语情境中出现的，AI 被放置到什么实践位置，以什么方式介入，为什么被评价，以及边界如何被生成。

## 五轴定义

### 1. 话语情境 `discursive_context / S`

话语情境描述材料在平台上的表达体裁和互动位置。它区分自我实践叙述、求助诊断、教程建议、规范争议、制度规则转述、工具服务推广和同行经验协商。

这个轴回答：用户是在展示做法、求助判断、教别人使用、争论规范、转述制度、推广工具，还是与同行协商经验。

### 2. 实践位置 `practice_position / P`

实践位置描述 AI 被放进科研活动的哪个位置。它包括选题定位、文献知识组织、研究设计、数据材料、分析建模、结果核验解释、写作发表传播、组织治理评价和学习训练。

这个轴回答：AI 介入的是科研产出、科研治理、科研训练，还是这些过程中的具体位置。

### 3. 介入方式 `intervention_mode / I`

介入方式描述 AI 在该实践位置上做了什么。它包括检索定位、摘要整理、草稿生成、润色翻译、分析计算、规划拆解、检查质控、自动化替代和陪练支持。

这个轴回答：AI 是辅助定位信息、组织材料、生成内容、优化表达、执行分析、规划流程、做质控、替代人完成任务，还是作为训练对象。

### 4. 规范评价 `normative_evaluation / N`

规范评价描述用户如何判断某种 AI 介入是否合理。它包括未评价、效率正当化、条件接受、可靠性担忧、原创/贡献担忧、诚信合规否定、责任披露要求、训练价值担忧和公平/隐私/资源风险。

这个轴回答：用户为什么接受、限制、担忧或否定某种 AI 科研使用。

### 5. 边界生成 `boundary_generation / G`

边界生成描述讨论如何把评价转化为使用规则。它包括辅助/替代分界、人类最终责任、任务风险分层、人工复核门槛、披露说明边界、高风险禁止、训练阶段保护、数据/知识资产边界和制度/期刊/学科规则边界。

这个轴回答：讨论最终划出了什么边界，以及这些边界依赖什么规则来源。

## 与旧框架的关系

旧框架中的科研工作流被拆解并重组为 `practice_position`；合法性评价被扩展为 `normative_evaluation`；边界协商被改写为 `boundary_generation`。v2 新增 `discursive_context` 与 `intervention_mode`，用于捕捉平台话语形式和 AI 实际介入机制。

因此，旧框架不再作为当前活跃主框架；它只保留为 `quality_v4 historical audit`、legacy mapping 和旧 artifacts 解读背景。

## 当前基线约束

本理论升级不改变当前正式数据口径：

- `quality_v5 post-only`
- 正式帖子 `514`
- 正式评论 `0`
- `comment_review_v2 deferred`
- `quality_v4 historical audit`

下一轮若要把 v2 接入 artifacts，需要单独升级 canonical schema、review 规则、reporting、figures 和 build 链路。本轮只建立理论和 codebook 基础。
