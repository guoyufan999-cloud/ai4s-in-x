# 第三章 研究设计（framework_v2 工作稿）

> 本稿用于衔接 `quality_v5 post-only` 工程链路、framework_v2 编码审计与论文方法章节。当前正式基线为 `514` 条正式帖子、`0` 条正式评论；`comment_review_v2` 仍为 deferred，不作为本文正式结果。

## 3.1 研究思路

本文关注的不是抽象层面上“AI 能否用于科研”，而是社交媒体用户如何在具体科研活动中呈现 AI 的使用方式，并围绕这种使用形成规范评价和边界协商。基于这一问题意识，本文将旧有的“科研工作流环节识别、合法性评价识别、边界协商机制识别”三线框架，升级为“话语情境 -> 实践位置 -> 介入方式 -> 规范评价 -> 边界生成”的五层分析框架。

这一框架的基本逻辑是：首先把帖子放回社交媒体话语情境中，识别其文本类型、表达身份和互动形式；其次判断 AI 被放置在科研生产、科研训练与能力建构、科研治理中的哪一类实践位置；再次识别 AI 在具体任务中的介入方式和介入强度；随后分析用户围绕效率、可靠性、原创性、责任、披露、诚信、公平等标准形成的规范评价；最后观察这些评价如何进一步转化为“合理辅助”与“不可接受替代”之间的边界。

因此，本文的分析不是单纯统计工具名称，也不是把平台态度压缩为支持或反对，而是把“AI 做什么”“做到什么程度”“为什么被认为可接受或不可接受”“哪些使用边界被生成”放在同一条分析链中处理。

## 3.2 数据来源与平台选择

本文数据来自公开社交媒体平台中关于 AI 介入科研活动的帖子层材料。当前正式论文口径使用 `quality_v5 post-only formal baseline`，即 `514` 条正式帖子和 `0` 条正式评论。评论层材料在工程上保留为后续可能扩展的对象，但 `comment_review_v2` 当前处于 deferred 状态，因此本文不把评论语料写成正式分析结果，也不使用历史评论统计替代当前论文结论。

选择社交媒体帖子作为研究对象，是因为这类材料能够呈现正式政策文本和问卷访谈之外的实践性表达。用户往往不会以完整制度话语讨论 AI 与科研，而是通过经验分享、教程展示、风险提醒、规范解读等形式，把 AI 嵌入文献阅读、选题构思、数据分析、论文写作、投稿评审、学术训练等具体任务。正是在这些日常化表达中，AI 介入科研活动的可接受性、风险和边界被不断重述。

需要说明的是，本文不把该平台样本视为全部科研共同体的代表性样本。它更适合回答“社交媒体中如何呈现和协商 AI 科研实践”这一问题，而不是推断所有研究者的总体态度分布。

## 3.3 样本获取与关键词设置

本研究基于既有研究工程链路形成样本，而非在本轮重新扩张采集。工程链路包括历史公开数据迁移、研究型主库构建、review queue、reviewed import、`quality_v5` post-only formal baseline 和 artifacts build。样本进入正式分析前，需要经过帖子层筛选、人工 reviewed 导入和质量基线检查。

关键词设置遵循宽义 AI4Research 口径，而非狭义 AI4S 口径。也就是说，本文关注 AI 进入科研生产、科研训练与能力建构、科研治理等活动场域，而不把研究对象限定为“AI 用于科学发现”或“AI for Science”中的技术性科学发现任务。旧项目中使用的 AI4S 表述在本文中作为历史项目名称和研究来源保留，正式论文题目与分析口径改为“社交媒体中 AI 介入科研活动的实践呈现、规范评价与边界协商研究”。

论文定稿时，关键词清单应以工程仓库中的采集与筛选记录为准，避免在方法章节中补写无法追溯的关键词。当前章节只确认本轮未新增平台抓取、未启动评论层正式编码，也未改变 `quality_v5` 的正式样本边界。

## 3.4 分析单位

本文的主要分析单位包括两个层次。第一层是帖子，即进入 `quality_v5 post-only` 正式范围的 `514` 条帖子。帖子层用于分析话语情境、文本类型、发帖者自我呈现、实践场域和工作流分布。

第二层是 claim unit，即帖子内部可识别的实践或评价单元。由于少数帖子同时包含多个实践判断，当前 framework_v2 审计报告识别到 `516` 个带有 v2 字段的 claim units。claim unit 层用于承载 A/B/C/D 旧框架字段，以及 F/G/H/I/J/K 新增 v2 字段。这样既保留了旧框架的可追溯性，也允许本文在更细粒度上分析 AI 介入方式、介入强度、评价张力、正式规范参照、边界协商机制和边界结果。

评论不是本文当前正式分析单位。将评论层排除在正式结果之外，是工程基线中的设计选择，不是导入遗漏。

## 3.5 编码框架

本文编码框架由旧 A/B/C/D 代码组和新增 F/G/H/I/J/K 代码组共同构成。

| 分析层次 | 代码组或字段 | 功能 |
|---|---|---|
| 话语情境 | platform、speaker_position_claimed、discursive_mode、record_type 等映射字段 | 描述平台文本如何呈现 AI 科研实践 |
| 实践位置 | A 组 | 区分科研生产、科研治理、科研训练与能力建构 |
| 介入方式 | F 组 | 区分信息辅助、生成辅助、分析建模、判断建议、自动执行、治理监督 |
| 介入强度 | G 组 | 区分低强度辅助、中强度共创、高强度替代 |
| 规范评价倾向 | B 组 | 区分未明确评价、接受、条件接受、拒斥、冲突/摇摆等倾向 |
| 规范评价标准 | C 组 | 识别效率、能力、可靠性、原创性、责任、披露、诚信、公平等评价依据 |
| 评价张力 | H 组 | 识别效率与原创性、便利性与可靠性、人机共创与责任归属等张力 |
| 正式规范参照 | I 组 | 识别期刊政策、学校规定、署名规则、披露要求、审稿规范等参照 |
| 边界类型 | D 组 | 保留辅助/替代、人机分工、责任、原创性、披露、诚信等边界内容 |
| 边界协商机制 | J 组 | 区分条件化、责任化、规范化、风险化机制 |
| 边界协商结果 | K 组 | 区分辅助合法化、条件合法化、风险问题化、替代去合法化等结果 |

其中，“合法性”不再作为唯一主术语，而是作为历史术语和理论来源被操作化为“规范评价”。这使本文能够避免把多维评价压缩为单一的合法/不合法判断。

## 3.6 编码流程与质量控制

编码流程分为四步。

第一步，形成 `quality_v5 post-only` 正式基线。该基线锁定 `514` 条正式帖子和 `0` 条正式评论，作为本文当前论文材料的唯一正式样本范围。`quality_v4` 只作为历史审计快照，不作为当前正式结果。

第二步，保留旧 A/B/C/D 字段。旧框架中的科研工作流、规范评价倾向、规范评价标准和边界类型继续作为可追溯字段存在，避免因框架升级造成历史编码不可比。

第三步，进行 framework_v2 补码。补码字段包括 F/G/H/I/J/K 六组，存储在 `reviewed_records.payload_json` 与 canonical JSONL payload 中，不进行数据库 schema migration。导入时只允许新增或修订 v2 字段，不允许改动既有 A/B/C/D 字段；若旧字段被改动，reviewed import 会拒绝导入。

第四步，构建 framework_v2 materials 与 coding audit。当前 `framework_v2_coding_audit_report` 显示：正式帖子覆盖 `514/514`，missing posts 为 `0`，v2 claim units 为 `516`，机械一致性检查状态为 `ok`，字段规则违规数为 `0`，provenance 标记缺失数为 `0`。审计规则包括 F 至少一个、G 正好一个、I 至少一个、I0 不与 I1-I8 混用、有 D 组边界时必须有 J/K、无 D 组边界时 J/K 必须为空、导入记录保持 reviewed 状态。

需要保留方法边界：F/G/H/I/J/K 字段来自“用户授权接受 AI 辅助编码草稿并保留 provenance”的补码流程。它可以支持工程一致性、覆盖完整性和描述性统计，但不能在论文中写成“双人独立人工编码”或“编码完全无误”。如果后续需要更强的方法说服力，应从高风险复核队列中抽取样本进行人工复核，并记录修正率与修正类型。

## 3.7 研究伦理

本文只使用公开可见的社交媒体帖子层材料，并在正式分析中避免暴露可识别个人身份的信息。引用或摘录时应优先使用去标识化文本，避免呈现账号名、链接、联系方式、头像、具体学校或可追踪个人身份的组合信息。

研究伦理还体现在样本边界控制上。本文不在本轮继续抓取平台数据，不启动评论层正式结果，不把历史评论 corpus 写成当前正式分析，不把 `quality_v4` 历史审计基线写成当前活跃正式口径，也不自动填充正式人工编码字段。对 AI 辅助编码过程的使用，应在方法章节中如实说明其 provenance 和人工接受边界。

综上，本文的方法可信度主要来自三个层面：第一，正式样本边界清晰且可复现；第二，旧框架和新框架之间存在可追溯映射；第三，framework_v2 补码经过 reviewed import 安全校验和 coding audit 机械一致性检查。其限制也同样明确：当前结果是社交媒体帖子层描述性分析，不代表全部科研共同体，也不包含正式评论层结论。
