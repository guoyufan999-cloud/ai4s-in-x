# 研究简报

## 一、研究背景

生成式 AI 已经进入科研实践，但平台讨论并不只是“是否使用 AI”或“是否提效”的意见集合。小红书中的 AI4S 内容同时包含自我经验、求助诊断、教程建议、规范争议、制度转述、工具推广和同行协商。这些材料更适合被理解为一种社交媒体话语场：用户在其中说明 AI 被放进什么科研位置、以什么方式介入、如何被评价，以及边界如何被生成。

因此，本项目的研究框架从旧的“科研工作流—合法性评价—边界协商”升级为 v2 五轴框架：“话语情境—实践位置—介入方式—规范评价—边界生成”。旧框架仍作为历史审计和 legacy 映射背景保留，但不再是当前活跃理论框架。

## 二、当前基线边界

本轮框架升级不改变任何正式数据事实：

- 当前正式基线：`quality_v5 post-only`
- 正式帖子 / 正式评论：`514 / 0`
- `comment_review_v2 deferred`
- `quality_v4 historical audit`
- 不修改 archive、database schema、artifacts build 或 outputs 中已有正式 artifacts

`formal_comments=0` 是本轮 post-only 设计选择，不是导入遗漏。评论层正式结果只有在后续单独启动 `comment_review_v2` 后才能进入论文主结果。

## 三、核心研究问题

1. 平台讨论在什么话语情境中呈现 AI4S：自我实践、求助、教程、争议、规则转述、推广或同行协商？
2. AI 被放置在科研实践的哪些位置：选题、文献、设计、数据、分析、核验、写作发表、组织治理或学习训练？
3. AI 以什么方式介入科研活动：检索、整理、生成、润色、分析、规划、质控、自动化替代或陪练支持？
4. 用户如何评价这些介入方式：效率正当化、条件接受、可靠性担忧、原创/贡献担忧、诚信合规否定、责任披露要求、训练价值担忧或公平/隐私/资源风险？
5. 这些评价如何生成边界：辅助/替代、人类最终责任、任务风险分层、人工复核、披露说明、高风险禁止、训练阶段保护、数据/知识资产或制度/学科规则边界？

## 四、研究目标

### 研究总目标

解释社交媒体中 AI4S 讨论如何从具体话语情境出发，把 AI 放置到不同科研实践位置，并通过介入方式、规范评价和边界生成来建构“可接受使用”的条件。

### 具体目标

1. 建立五轴 codebook，替代旧三轴主框架。
2. 区分平台话语情境，避免把教程、求助、推广和规范争议混为同一种材料。
3. 细化 AI 的实践位置与介入方式，避免只按宏观科研流程计数。
4. 将规范评价与边界生成拆开，分别观察“为什么被评价”和“边界如何被形成”。
5. 为后续 artifacts v2 升级预留清晰接口，但本轮不重建 artifacts。

## 五、研究对象

本研究对象是公开平台中与 AI4S 科研实践、评价和边界相关的帖子文本及其可审计元信息。当前正式论文口径只使用 `quality_v5 post-only` 的帖子层结果；评论语料和历史评论分析保留为后续工作或审计背景，不能写成当前正式评论结果。

## 六、方法路径

本轮方法路径只更新理论和编码层：

1. 以 v2 五轴重写理论框架和论文大纲。
2. 更新 codebook 和 coding rules。
3. 将 `iter_codebook_rows()` 的主输出切到五轴代码。
4. 保留 legacy lookup helper，避免破坏旧导入和历史审计。
5. 新增文档和 codebook 测试，确认 v2 是当前活跃框架。

数据库 schema、canonical JSONL schema、review/prefill/rescreen 规则、reporting、figures 和 artifacts build 均留到下一轮。
