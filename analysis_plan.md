# 分析计划

## 一、整体研究流程

本项目按以下研究流程推进：

1. 历史公开讨论材料迁移入研究主库
2. 数据去重、字段标准化、时间标准化与匿名化
3. 帖子与评论的分析样本确认
4. 围绕工作流、AI 实践方式、合法性判断和边界协商机制进行编码
5. 生成描述统计与交叉比较视图
6. 输出论文表格、图表、摘录和结果叙述

## 二、数据处理步骤

### 1. 数据迁移

- 输入：legacy SQLite 运行库、正式冻结产物、查询词模板
- 输出：`data/processed/ai4s_legitimacy.sqlite3`
- 关键要求：
  - 原始记录不丢失
  - 个人标识哈希化或掩码化
  - 明确记录迁移批次与来源

### 2. 去重与标准化

- 帖子以 `post_id / legacy_note_id / canonical_url` 为主标识
- 评论以 `comment_id` 为主标识
- 日期统一为 `YYYY-MM-DD`
- 文本统一做空白字符规范化
- 查询词与来源说明进入辅助表

### 3. 匿名化

- 作者 ID 统一哈希
- 评论者 ID 统一哈希
- 作者名称与评论者名称尽量掩码化
- 不在研究主库保留不必要的身份识别信息

## 三、编码步骤

### 1. 工作流环节编码

目标是识别讨论涉及的科研环节，例如选题、文献、研究设计、数据处理、建模、写作、投稿、协作管理等。

### 2. AI 实践方式编码

目标是识别 AI 是以替代执行、辅助建议、自动生成、结构化整理、质量检查、解释支持还是协作协调的方式介入。

### 3. 合法性判断编码

目标是识别讨论中支持或反对 AI 介入的核心判断标准，例如效率正当性、学术诚信、原创性、可解释性、可复现性、责任归属、学科规范一致性等。

### 4. 边界协商机制编码

目标是识别人们如何划分“可以接受的辅助”与“不可接受的替代”，以及如何讨论学生、科研人员、作者署名、人机分工等边界。

## 四、合法性分析维度

本研究中的合法性分析至少包括以下维度：

- 效率正当性
- 专业能力边界
- 原创性
- 学术诚信
- 可解释性
- 可复现性
- 责任归属
- 工具适配性
- 学科规范一致性
- 教育/训练价值

## 五、比较分析维度

当前分析优先围绕以下维度进行比较：

1. 时间维度：月度、季度、半年度
2. 工作流维度：不同科研环节
3. 样本层级：帖子与评论
4. 态度维度：支持、保留、担忧、拒绝等判断取向
5. 合法性维度：不同判断标准的出现频率与组合关系
6. 边界维度：不同人群、不同学科、不同实践方式之间的边界协商差异

## 六、输出指标

本项目优先输出以下指标：

- 正式研究样本规模
- 工作流环节分布
- AI 实践方式分布
- 合法性判断维度分布
- 评论态度结构
- 工作流 × 合法性维度交叉情况
- 边界协商机制的主题汇总

## 七、结果呈现形式

当前正式结果呈现形式包括：

- 研究型主库和分析视图
- 投稿版图表包
- 投稿版结果章节、方法与限制章节
- 差异报告（如 `quality_v3 -> quality_v4`）
- 论文正文引用用的摘录与表格

本轮重构后，所有正式输出都统一进入 `outputs/`，不再混放在历史运行导出目录中。
