# 分析计划

## 一、整体研究流程

当前项目进入 framework v2 第一轮：只升级理论框架、论文大纲、codebook、任务文档与测试。正式数据链仍保持 `quality_v5 post-only`，正式帖子 `514`、正式评论 `0`，`comment_review_v2 deferred`，`quality_v4 historical audit`。

本轮不执行：

- database schema 修改
- artifacts build 修改
- 数据重跑或数据库导入
- outputs 正式 artifacts 改写
- archive 调整

## 二、v2 分析框架

v2 框架由五个编码轴组成。

### 1. 话语情境 `discursive_context / S`

识别材料以什么话语形式出现：自我实践叙述、求助诊断、教程建议、规范争议、制度规则转述、工具服务推广或同行经验协商。这个轴用于避免把平台上的不同表达体裁混成同一种“态度”。

### 2. 实践位置 `practice_position / P`

识别 AI 被放置在哪个科研实践位置：选题定位、文献知识组织、研究设计、数据材料、分析建模、结果核验解释、写作发表传播、组织治理评价或学习训练。

### 3. 介入方式 `intervention_mode / I`

识别 AI 以什么方式介入：检索定位、摘要整理、草稿生成、润色翻译、分析计算、规划拆解、检查质控、自动化替代或陪练支持。

### 4. 规范评价 `normative_evaluation / N`

识别用户如何评价 AI 介入：未评价、效率正当化、条件接受、可靠性担忧、原创/贡献担忧、诚信合规否定、责任披露要求、训练价值担忧或公平/隐私/资源风险。

### 5. 边界生成 `boundary_generation / G`

识别讨论如何生成边界：辅助/替代分界、人类最终责任、任务风险分层、人工复核门槛、披露说明边界、高风险禁止、训练阶段保护、数据/知识资产边界或制度/期刊/学科规则边界。

## 三、编码步骤

1. 先判断话语情境：材料是在讲自己的实践、求助、给教程、争规范、转述制度、推广工具，还是进行同行协商。
2. 再判断实践位置：AI 被放进哪类科研位置。
3. 再判断介入方式：AI 是检索、整理、生成、润色、分析、规划、质控、替代还是陪练。
4. 再判断规范评价：文本是否正当化、条件接受、担忧、否定或要求披露/责任。
5. 最后判断边界生成：文本是否实际划出了使用边界。

允许多重编码，但每个代码都必须能回到原文证据。不能为了凑满五轴而补码。

## 四、纳入与剔除

### 纳入标准

- 明确涉及 AI 或可识别 AI 工具
- 能回到科研实践、科研治理、科研传播或科研训练语境
- 至少能支持五轴中的一个实质判断

### 默认剔除

- 泛化科技新闻、纯产品广告、普通办公学习、求职编程、低信息口号
- 只有 AI 热点，没有科研实践位置
- 无法提供任何可审计原文证据的材料

## 五、比较分析维度

本轮只定义下一轮可用的分析维度，不生成新 artifacts：

1. 话语情境 × 实践位置
2. 实践位置 × 介入方式
3. 介入方式 × 规范评价
4. 规范评价 × 边界生成
5. 时间 × 五轴分布
6. 学科/主体类型 × 五轴分布

## 六、当前输出边界

当前正式输出仍以既有 `quality_v5 post-only` artifacts 为准。本轮新增文档和测试只说明 framework v2 的理论与编码设计，不把旧评论层分析恢复为当前结果，也不把 `quality_v4` 写回 active baseline。

## 七、下一轮 artifacts 升级入口

如果下一轮将 v2 接入正式 artifacts，应另起任务修改 canonical constants/schema、claim units、review/prefill/rescreen 规则、external XHS coding、review_v2 artifacts、reporting、figures queries、build scripts，必要时再处理 database views template。本轮全部不动。
